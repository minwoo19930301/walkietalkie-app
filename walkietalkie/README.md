# 워키타키

둘만 쓰는 링크 기반 영상·음성 통화 앱입니다. 설치형 앱 대신 웹으로 만들고, 서버는 연결만 붙여주는 최소 역할만 맡겨서 유지비를 거의 0원에 가깝게 맞추는 방향으로 구성했습니다.

## 링크

- GitHub 저장소: [https://github.com/minwoo19930301/walkietalkie-app](https://github.com/minwoo19930301/walkietalkie-app)
- 현재 배포 주소: [https://walkietalkie.rlaalsdn456456.workers.dev/](https://walkietalkie.rlaalsdn456456.workers.dev/)
- 이전 배포 주소: [https://paircall.rlaalsdn456456.workers.dev/](https://paircall.rlaalsdn456456.workers.dev/)

## 왜 웹 기반이 맞는가

- 아이폰, 안드로이드, 노트북에서 앱 설치 없이 같은 URL로 바로 접속할 수 있습니다.
- 미디어는 WebRTC로 브라우저끼리 직접 전송하므로 서버가 영상 데이터를 계속 중계하지 않습니다.
- 정적 파일 + Cloudflare Worker만으로 배포가 가능해서 운영이 단순합니다.

## 구조

- 프런트엔드: 정적 HTML/CSS/JS
- 신호 서버: Cloudflare Worker + Durable Object
- 통화 전송: WebRTC P2P
- 데이터 저장: 없음
- 회원가입/계정: 없음

초대 링크는 URL 해시(`#room=...&key=...`)에 개인 토큰을 담고, 브라우저가 그 값을 SHA-256으로 해시한 뒤 방 ID를 만듭니다. 따라서 원본 개인 토큰은 서버 요청 경로로 직접 전달되지 않습니다.

## 서버 비용

기본 운영은 다음 기준으로 거의 0원에 가깝습니다.

- 호스팅: Cloudflare Workers Free
- 기본 도메인: `*.workers.dev` 사용 시 추가 비용 없음
- 저장소/DB: 사용 안 함
- 미디어 릴레이: 기본값은 사용 안 함

즉, `커스텀 도메인을 따로 사지 않고`, `TURN 서버를 추가하지 않으면` 고정 서버 비용 없이 계속 운영 가능합니다.

다만 이건 `영상과 음성이 브라우저끼리 직접 오가는 P2P` 구조라서 가능한 것입니다. 서버는 영상 데이터를 계속 들고 있지 않고 처음 연결만 붙여줍니다.

## 중요한 제약

이 프로젝트는 기본값이 `STUN only` 입니다. 즉, 대부분의 일반 네트워크에서는 바로 붙지만, 일부 이동통신망/회사망/공용 와이파이 조합에서는 직접 연결이 실패할 수 있습니다.

그 경우 선택지는 둘입니다.

1. 그대로 쓰고, 연결이 안 되는 네트워크에서는 다른 와이파이나 LTE/5G로 바꿔서 사용
2. 나중에 TURN 릴레이를 추가해서 연결 안정성을 높임

비용 0원을 우선한 MVP라서 기본값은 1번 전략입니다.

## 로컬 실행

```bash
npm install
npm run dev
```

Wrangler가 로컬 개발 서버를 띄우고, 정적 자산과 Durable Object를 함께 실행합니다.

## 배포

Cloudflare 계정 로그인이 되어 있다면:

```bash
npm run deploy
```

배포 후 발급되는 `workers.dev` 주소 하나만 있으면 됩니다. 그 주소를 두 사람이 북마크해서 계속 쓰면 됩니다.

## 추천 운영 방식

- 한 번 배포
- 첫 접속 시 생성된 개인 링크를 두 사람만 저장
- 이후엔 항상 그 링크로 접속
- 필요할 때만 `통화 시작`
